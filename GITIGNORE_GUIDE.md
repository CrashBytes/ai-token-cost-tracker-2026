# .gitignore - Tutorial-Optimized Configuration

## 📋 Overview

This `.gitignore` is optimized for the **AI Token Cost Tracker Chrome Extension Tutorial** on CrashBytes.

## ✅ What's COMMITTED (Users get these files)

These files are in the repository to help you complete the tutorial:

| File/Folder | Purpose | Why Committed |
|-------------|---------|---------------|
| `package-lock.json` | Dependency versions | Ensures exact same versions as tutorial |
| `jest.config.js` | Test configuration | Pre-configured for 100% coverage |
| `__tests__/` | Complete test suite | 143 tests ready to run |
| `__mocks__/` | Chrome API mocks | Testing infrastructure |

## ✅ What Users Should Commit (in their forks)

If users extend the tutorial, they should commit:

✅ **Source code changes** (`lib/`, `background/`, `popup/`, etc.)
✅ **New tests** (in `__tests__/`)
✅ **Updated documentation** (README updates)
✅ **Package.json changes** (new dependencies)

---

## 🚫 What Users Should NEVER Commit

❌ **node_modules/** - Always regenerate with `npm install`
❌ **coverage/** - Always regenerate with `npm test --coverage`
❌ **dist/** - Always regenerate with `npm run build`
❌ **.pem files** - Private keys (security risk!)
❌ **.env files** - May contain API keys
❌ **Personal notes** - Keep local

---

## 🔍 Verification

To verify .gitignore is working:

```bash
# Check what's tracked
git status

# See what's ignored
git status --ignored

# Test if file would be ignored
git check-ignore -v filename.txt
```

## 📝 Best Practices for Tutorial Users

### **DO:**
✅ Run `npm install` to get dependencies
✅ Run `npm test` to verify everything works
✅ Create your own notes in `notes.md` (it's ignored)
✅ Take screenshots in `screenshots/` folder (it's ignored)
✅ Experiment with personal config overrides

### **DON'T:**
❌ Commit `node_modules/` - waste of space
❌ Commit `coverage/` - regenerated on each test run
❌ Commit `.pem` files - security risk
❌ Commit personal notes to upstream
